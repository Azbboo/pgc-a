# PGC Dashboard 视觉系统与组件规范设计

日期：2026-05-03

## 1. 设计目标

PGC Dashboard 是交易执行辅助工具，不是资讯门户，也不是展示型大屏。

视觉系统必须服务于三件事：

1. 快速判断今天有没有动作；
2. 避免把信号、计划、成交、持仓、Agent 意见混淆；
3. 在数据异常、账户风险、T+2/T+5 到期时给出强提醒。

设计取向：

- 安静、密集、专业；
- 信息优先，装饰克制；
- 适合每天重复使用；
- 对危险操作足够谨慎；
- 页面像交易工作台，而不是营销 Dashboard。

明确禁止：

- 大面积装饰渐变；
- 首屏 Hero；
- 花哨宣传文案；
- 卡片套卡片；
- 把 Agent 意见做成“买卖指令”样式；
- 用单一颜色家族撑完整个系统；
- 使用模糊的“今日/明日”替代明确日期。

## 2. 视觉原则

### 信息密度

默认采用高密度工作台布局。

规则：

- 列表页以表格为主；
- 关键动作页使用左右分栏；
- 指标区只放当天决策需要的信息；
- 历史解释放抽屉或详情页；
- 不把每个指标都做成大卡片；
- 卡片只用于独立对象，如一只候选、一条计划、一笔持仓。

### 视觉层级

层级顺序：

1. 数据质量 blocker；
2. 今日/明日必须处理动作；
3. 当前 active 计划；
4. 当前持仓 T+2/T+5；
5. Agent 复核意见；
6. 历史信号和研究说明。

页面上任何时候 blocker 的视觉优先级都高于收益表现。

### 日期明确性

所有交易相关日期必须显示绝对日期。

允许：

- `复盘日 20260430`
- `计划买入 20260506`
- `T+2 20260508`

不允许单独显示：

- `今天`
- `明天`
- `后天`
- `两天后`

可以组合显示：

- `下一交易日 20260506`
- `T+2 20260508`

## 3. 颜色语义

颜色用于语义，不用于装饰。

| 用途 | 语义 | 建议色 |
| --- | --- | --- |
| 页面背景 | 工作台底色 | `#F7F8FA` |
| 主内容背景 | 面板/表格 | `#FFFFFF` |
| 主文字 | 高对比文本 | `#111827` |
| 次级文字 | 辅助信息 | `#6B7280` |
| 边框 | 分隔线 | `#E5E7EB` |
| 主操作 | 发布/运行/保存 | `#2563EB` |
| 正收益/止盈 | 正向状态 | `#059669` |
| 风险/止损/blocker | 阻断风险 | `#DC2626` |
| 警告/人工复核 | 谨慎状态 | `#D97706` |
| 中性/跳过/过期 | 中性状态 | `#64748B` |
| Agent | 复核意见 | `#4F46E5` |

使用规则：

- 红色只用于 blocker、止损、严重错误、危险操作；
- 绿色只用于真实或模拟持仓收益、止盈结果；
- 蓝色只用于主操作和可点击导航；
- 紫/靛色只用于 Agent 复核，不用于策略信号；
- 灰色用于跳过、过期、无信号；
- 同一页面最多同时突出一个主按钮。

## 4. 字体与排版

首版建议使用系统可用字体，不引入远程字体依赖。

字体栈：

```css
font-family:
  ui-sans-serif,
  -apple-system,
  BlinkMacSystemFont,
  "Segoe UI",
  "PingFang SC",
  "Microsoft YaHei",
  sans-serif;
```

字号：

| 用途 | 大小 | 行高 |
| --- | --- | --- |
| 页面标题 | 20px | 28px |
| 区块标题 | 15px | 22px |
| 表格正文 | 13px | 20px |
| 辅助文本 | 12px | 18px |
| 数字指标 | 18px | 26px |
| 状态 Chip | 12px | 18px |

规则：

- 不使用视口宽度缩放字体；
- 数字使用 tabular numbers；
- 表格内文本不使用超大字号；
- 状态文案最多两行，超出折叠；
- 金额、价格、收益率右对齐。

## 8. 组件规范

### DataStatusBar

用途：每个页面顶部统一展示运行上下文。

字段：

- `account_key`
- `account_type`
- `review_date`
- `latest_market_date`
- `strategy_version`
- `open_positions/max_positions`
- `data_quality_status`

状态：

- normal；
- warning；
- blocker。

交互：

- 点击账户打开账户切换；
- 点击复盘日打开日期选择；
- 点击数据质量跳转数据质量页。

### ActionPanel

用途：每日复盘页显示最重要的下一步动作。

状态：

| 状态 | 主信息 | 主按钮 |
| --- | --- | --- |
| 无信号 | 无策略信号 | 只读 |
| 有买入计划 | 股票 + 计划交易日 | 发布计划 |
| 仓位已满 | 有信号但无空位 | 查看持仓 |
| 有卖出计划 | 持仓 + 退出原因 | 录入卖出成交 |
| blocker | 阻断原因 | 处理数据质量 |

规则：

- 一个 ActionPanel 只能有一个主按钮；
- blocker 状态隐藏普通主按钮；
- 没有成交时不显示“已买入”。

### EntityTable

用途：交易计划、当前持仓、数据质量、Agent 复核列表。

规范：

- 表头固定；
- 支持排序；
- 支持筛选；
- 支持列密度切换；
- 行点击打开右侧详情抽屉；
- 行内只保留高频动作；
- 长文本截断，详情放抽屉。

### DetailDrawer

用途：展示血缘、JSON、事件、成交链路。

布局：

```text
详情抽屉
├── 标题 + 状态 Chip
├── 关键字段摘要
├── Tabs
│   ├── 详情
│   ├── 血缘
│   ├── JSON
│   └── 事件
└── 操作区
```

规则：

- 抽屉宽度桌面 520px 到 720px；
- JSON 默认折叠；
- 危险操作必须二次确认；
- 抽屉内不再嵌套卡片。

### ConfirmDialog

用途：发布、取消、修正、冲销、人工覆盖。

必须显示：

- 账户；
- 股票；
- 日期；
- 动作；
- 影响对象 ID；
- 是否会写事实表；
- 操作者；
- 原因输入框，危险操作必填。

### LineageTrail

用途：展示血缘链路。

示例：

```text
raw_event #207
 → feature_snapshot #801
 → strategy_signal #123
 → daily_pick #41
 → trade_plan #29
 → trade #51
 → position #18
```

规则：

- 缺失环节要显示为“未生成”，不能隐藏；
- Agent 是旁路引用，使用虚线视觉；
- Report 不进入事实链路。

### JsonViewer

用途：

- `features_json`
- `plan_json`
- `payload_json`
- `raw_decision_json`
- `domain_events.payload_json`

规则：

- 默认只展开前两层；
- 搜索字段；
- 高亮禁止字段检测结果；
- 支持复制字段路径；
- 不展示 token。

## 9. 表单规范

### 成交表单

买入字段顺序：

1. 计划；
2. 账户；
3. 股票；
4. 成交日期；
5. 成交价；
6. 股数；
7. 手续费；
8. 税费；
9. 来源；
10. 备注。

卖出字段顺序：

1. 持仓；
2. 卖出计划；
3. 账户；
4. 股票；
5. 成交日期；
6. 成交价；
7. 卖出股数；
8. 手续费；
9. 税费；
10. 备注。

字段校验：

| 字段 | 校验 |
| --- | --- |
| 成交日期 | 必须是交易日 |
| 成交价 | 大于 0 |
| 股数 | 大于 0 |
| 买入股数 | 建议 100 股整数倍 |
| 卖出股数 | 不超过当前持仓 |
| 账户 | 必须和计划/持仓一致 |
| live 来源 | 只能 manual 或 broker_import |
| 修正原因 | 必填 |

### 危险操作表单

危险操作：

- 取消 active 计划；
- 冲销成交；
- 修正成交；
- 人工覆盖退出；
- 忽略 blocker；
- 暂停策略版本。

要求：

- 二次确认；
- 原因必填；
- 显示影响范围；
- 保存 domain event；
- 不允许静默删除。

## 11. 响应式规则

### 桌面端

宽度 `>= 1200px`：

- 左侧导航固定；
- 每日复盘可双栏；
- 表格列完整展示；
- 详情抽屉右侧滑出。

### 平板端

宽度 `768px - 1199px`：

- 左侧导航可收起为图标；
- 每日复盘上下堆叠；
- 表格隐藏低优先列；
- 抽屉宽度 80%。

### 移动端

宽度 `< 768px`：

- 导航为底部或顶部菜单；
- 表格切换为列表行；
- 只展示核心字段；
- 成交录入表单单列；
- 危险操作仍然二次确认；
- 不隐藏 blocker。

移动端核心字段：

- 每日复盘：动作、股票、计划日、数据质量；
- 交易计划：动作、状态、股票、计划日；
- 持仓：股票、收益、T+2/T+5、下一动作；
- 成交录入：成交价、股数、日期、保存。

## 13. 可访问性与键盘操作

基础要求：

- 所有按钮可键盘聚焦；
- 表格行可 Enter 打开详情；
- Esc 关闭抽屉或弹窗；
- 表单错误关联到字段；
- 状态颜色必须配文案；
- 不只用颜色表达盈亏和风险；
- 危险操作确认按钮文案必须明确动作。

快捷键建议：

| 快捷键 | 动作 |
| --- | --- |
| `g d` | 每日复盘 |
| `g p` | 交易计划 |
| `g t` | 成交录入 |
| `g h` | 当前持仓 |
| `/` | 聚焦搜索 |
| `Esc` | 关闭抽屉/弹窗 |

快捷键不应在首版页面显眼展示，可放在帮助菜单。

## 14. 空状态与加载状态视觉

### 空状态

| 场景 | 文案 |
| --- | --- |
| 无信号 | 该复盘日无策略信号 |
| 无待处理计划 | 当前没有待处理交易计划 |
| 无持仓 | 当前账户无未平仓持仓 |
| 无 Agent 运行 | 该复盘日尚未发起 AI 复核 |
| 无数据质量问题 | 当前门禁通过 |

空状态规则：

- 不使用宣传文案；
- 不占据过大高度；
- 提供下一步动作；
- 不把无信号写成 skipped trade plan。

### 加载状态

| 场景 | 展示 |
| --- | --- |
| 刷新行情 | 顶部栏显示进行中，禁用复盘按钮 |
| 运行复盘 | ActionPanel 显示进度 |

## 15. 错误提示规范

错误提示格式：

```text
错误码：MARKET_DATA_NOT_READY
原因：300077.SZ 缺少 20260430 日行情。
处理：请刷新行情或在数据质量页确认停牌状态。
```

错误提示必须包含：

- 错误码；
- 受影响对象；
- 日期；
- 下一步处理；
- 跳转入口。

错误提示禁止：

- 只显示“失败”；
- 吞掉后端错误；
- 自动重试写操作；
- 让用户猜是账户问题还是数据问题。

## 17. 设计验收标准

视觉与组件设计落地后必须满足：

1. 页面第一屏能看到账户、复盘日、最新行情日、策略版本、数据质量。
2. 每日复盘页能在 10 秒内判断明日是否有动作。
3. 信号、计划、成交、持仓、Agent 意见有明确视觉区隔。
4. open blocker 存在时，主操作不可用且原因可见。
5. 成交录入页不会让用户在没有 active 计划时误录 live 买入。
6. 当前持仓页不会显示未成交计划为持仓。
7. Agent reject 在 advisory 模式下不会表现为自动卖出/跳过。
8. 移动端不隐藏 T+2/T+5 和 blocker。
9. 表格长文本、按钮文案、状态 Chip 不溢出。
10. 所有危险操作都有二次确认和原因输入。

## 18. ADR

### ADR-UI-001: 采用高密度工作台，而不是大屏看板

Context：用户每天要做复盘、计划确认、成交录入和退出处理，需要快速扫描和准确操作。

Options：

- 大屏指标看板；
- 高密度工作台；
- 移动优先卡片流。

Decision：采用高密度工作台。

Consequences：

- 好处：适合重复操作和审计追踪。
- 代价：视觉上不如营销式 Dashboard 抢眼。
- 风险：需要控制密度，避免表格信息过载。

### ADR-UI-002: 使用语义色而不是装饰色

Context：交易系统里的颜色会影响用户判断。颜色必须表达状态，不能只是装饰。

Options：

- 品牌色主导；
- 多彩装饰；
- 语义色系统。

Decision：使用语义色系统。

Consequences：

- 好处：风险、收益、Agent、阻断状态清楚。
- 代价：视觉表达更克制。
- 风险：后续新增状态必须先定义语义，不能随意选色。

### ADR-UI-003: 详情信息放抽屉，事实链路用 LineageTrail

Context：主页面要快，但审计需要完整链路。

Options：

- 所有信息直接铺在页面；
- 每个对象独立详情页；
- 主表格 + 右侧抽屉 + 血缘链路。

Decision：采用主表格 + 右侧抽屉 + LineageTrail。

Consequences：

- 好处：主流程快，追溯也完整。
- 代价：抽屉组件要支持复杂内容。
- 风险：抽屉内不能再堆过多嵌套卡片。
